## 1. Linux 体系结构
![Linux 体系结构](/tool/linux-system.png) <br>
#### 1. 系统调用
系统调用时操作系统的最小功能单位。例如：用户态想要申请一块20K大小的动态内存，就需要brk系统调用，将数据段指针向下偏移，如果用户态多处申请20K动态内存，同时又释放呢？这个内存的管理就变得非常的复杂。
#### 2. 库函数
库函数就是屏蔽这些复杂的底层实现细节，减轻程序员的负担，从而更加关注上层的逻辑实现。
#### 3. shell
`Shell`，外壳的意思，一种特殊的应用程序，俗称命令行。

## 3. 常用命令
### 3.1 基本命令
#### 3.1.1 ls 
- **命令：** ls [选项] [文件或目录]
- **选项：**
    - -a 显示所有文件，包括隐藏文件
    - -l 显示详细信息
    - -d 查看目录本身的属性而非子文件
    - -h 人性化的显示文件大小

``` bash
ls -l
drwxr-xr-x . 1 root root 800 Sep 16 00:19 logs
```
| drwxr-xr-x     | .       | 1              | root   | root   | 800      | Sep 16 00:19 | logs   |
|:---------------|:--------|:---------------|:-------|:-------|:---------|:-------------|:-------|
| 文件类型和权限 | ACL权限 | 硬链接引用次数 | 所有者 | 所数组 | 文件大小 | 最后修改时间 | 文件名 |
#### 文件类型和权限
drwxr-xr-x
- d: 第一位表示文件类型。d是目录文件，l是链接文件，-是普通文件，p是管道
- rwx: 第2-4位表示这个文件的属主(user)拥有的权限，r是读，w是写，x是执行。
- r-x: 第5-7位表示和这个文件属主所在同一个组(group)的用户所具有的权限。
- r-x: 第8-10位表示其他用户(other)所具有的权限。

**读、写、运行三项权限可以用数字表示，就是r=4,w=2,x=1。所以，-rw-r--r--用数字表示成644。**

- **命令：** mv [源文件或者⽬目录] [⽬目标⽂文件]
#### 3.2.8 ln
链接命令，生成链接文件
- **命令：** ln [选项] [源文件] [目标文件]
- **选项**
    - -s 生成软链接
- **硬链接特征**
    - 拥有相同的i节点和存储block块，可以看作是同⼀个⽂件 可以通过i节点访问
    - 以文件副本的形式存在。但不占用实际空间
    - 不能跨分区
    - 不能针对目录使用
    - **一般不不使⽤**
- **软链接**
    - 以路径的形式存在,类似于windows的快捷方式
    - 软链接拥有⾃己的i节点和Block块，但是数据块中只保存源⽂件的⽂件名和i节点号，并没有实际的文件数据
    - lrwxrwxrwx 软链接的⽂件权限都是 777
    - 删除源文件，软链接不能使⽤
    - 软链接文件必须是绝对路径
### 3.3 文件搜索
#### 3.3.1 locate
- **命令：** locate [选项] [文件/目录]
- **选项：**
    - c 显示查找到的数量
    - r 使用正则
- **说明：**
    - 选项为目录时，递归显示目录下的文件
    - locate是数据库查找
        - 数据保存在 /var/lib/mlocate 后台数据库，每天更新⼀次
        - 可以使用/usr/bin/updatedb   主要用来更新数据库
#### 3.3.2 whereis
搜索命令所在路路径以及帮助⽂文档所在位置
- **命令：** whereis [选项] [命令名]
- **选项：** 
    - -b 只查找可执行文件
    - -m 只查找帮助文件
#### 3.3.3 which
查看命令的路径或者别名
#### 3.3.4 find
文件搜索
- **命令：** find [搜索范围] [搜索条件]
- **搜索条件：**
    - -name
        - 普通文件名  tmp.js
        - **通配符** 
            - `*` 匹配任意内容
            - `?` 匹配任意⼀个字符
            - `[]` 匹配任意一个中括号内的字符
    - -i 不区分大小写
    - -user 按所有者进行搜索
    - -nouser
    - -atime 文件访问时间的时间范围
    - -ctime 文件改变属性的时间范围
    - -mtime 修改文件内容的时间范围
    - -size 文件大小
        - k小写、M大写
        - -8k：小于8K；8k：等于8k；+8k：大于8k
    - -inum 按i节点搜索
    - 综合条件
        - -a 逻辑与
        - -o 逻辑或
        - - exec 对上个命令的结果进行操作

``` bash
# 5天前修改的/nginx/access.log
find /nginx/access.log -mtime +5

# 大于10k小于20k
find /tmp -size +10k -a -size -20k
```
#### 3.3.5 grep
文件中匹配字符串
- **命令：** grep [选项] [字符/正则] [文件/目录]
- **选项：**
    - -i 忽略大小写
    - -v 排除字符串，反向选择
- **注意**
    - find命令，在系统当中搜索符合条件的⽂件名，如果需匹配，使用通配符匹配，通配符是完全匹配
    - grep命令，在⽂件当中搜索符合条件的字符串，如果需要匹配，使⽤用正则表达式进行匹配，正则表达式时包含匹配

### 3.5 压缩与解压缩
#### 3.5.1 zip
- **命令：** zip [选项] [压缩文件] [源文件]
``` bash
# 压缩文件
zip 1.zip book/1.txt

# 压缩目录

```
#### 3.5.2 gzip
gzip不能对整个目录进行压缩，只能压缩目录下的文件
| 命令                          | 示例                     | 含义                               |
|:------------------------------|:-------------------------|:-----------------------------------|
| gzip [文件]                   | gzip 1.txt               | 压缩为.gz文件，源文件会丢失        |
| gzip -c [源文件] > [压缩文件] | gzip -c 1.txt > 1.txt.gz | 压缩文件，源文件不会丢失           |
| gzip -r 目录                  | gzip -r temp             | 压缩文件下所有子文件，文件夹不压缩 |
| gzip -d [压缩文件]            | gzip -d temp.js.gz       | 解压缩，不保留压缩文件             |
| gunzip [压缩文件]             | gunzip tem.js.gz         | 解压缩，不保留压缩文件             |
#### 3.5.3 bz2格式
bzip2不能压缩目录
| 命令                | 示例               | 含义                               |
|:--------------------|:-------------------|:-----------------------------------|
| bzip2 [源文件]      | bzip2 1.txt        | 压缩为.bz2格式的文件，不保留源文件 |
| bzip2 -k [源文件]   | bzip2 -k 1.txt     | 压缩为.bz2格式的文件，保留源文件   |
| bzip2 -d [压缩文件] | bzip2 -d 1.txt.bz2 | 解压缩                             |
| bunzip2 [压缩文件]  | bunzip2 1.txt.bz2  | 解压缩                             |

### 3.9 shell
- shell是一个命令行解释器，它为⽤用户提供了一个向Linux内核发送请求以便运行程序的界⾯系统级程序
- 用户可以用Shell来启动、挂起、停⽌或者编写一些程序 
- Shell还是一个功能相当强大的编程语言，易编写，易调试，灵活性较强。 
- Shell是解释执⾏的脚本语⾔，在Shell中可以直接调用Linux系统命令。

#### 3.9.3 管道符号
多命令执行顺序
- ;分号，没有任何逻辑关系的连接符。每条命令的执行成功与否都会执行下面的命令。
- || 逻辑或。
- && 逻辑与。
- | 管道符，前面命令的输出会交给后面的命令去处理；前面的命令执行失败会报错，后面的命令处理不了数据也会报错。

通配符(同正则)
| 通配符 | 作用                          |
|:-------|:------------------------------|
| ?      | 匹配任一字符                  |
| *      | 匹配0个或任意个字符           |
| []     | 匹配括号中任一个字符          |
| [-]    | 匹配括号中任一个字符,表示范围 |
| [^]    | 匹配不是括号中的任一字符      |
其他符号
| 符号 | 作用                                                                                    |
|:-----|:----------------------------------------------------------------------------------------|
| ''   | 单引号，单引号中的特殊符号都没有特殊意义，如$ ` 都没有特殊意义                          |
| ""   | 双引号，双引号中的特殊符号都没有意义， $  \  除外，拥有调用变量值，引用命令和转义的含义 |
| ``   | 反引号，里面是系统命令                                                                  |
| $()  | 和反引号一样                                                                            |
| #    | 注释                                                                                    |
| $    | 调用变量值                                                                              |
| \    | 转义                                                                                    |
