# 位运算

位运算是低级的运算操作，所以速度也是最快的（相对于加减乘除），借助位运算的特性实现一些算法有很多好处

## 1. 基本概念
### 1.1 整数
ECMAScript 中整数有两种类型
- 有符号整数（正数、负数）
    - 有符号整数使用 31 位表示整数的数值，用第 32 位表示整数的符号，0 表示正数，1 表示负数。
- 无符号整数（只允许正数）

**位运算会把二进制数限制在32位，超出部分会被舍弃**
### 1.2 源码、反码、补码
- **源码**：数字转换成二进制数，最左边表示符号位，0是正数，1是负数
- **反码**：正数的反码与源码相同，负数的反码除符号位，其它取反
- **补码**：正数的补码与源码相同，负数的补码取反码 +1

## 2. 位运算符
| 运算符     | 用法  | 描述                                                      |
|:-----------|:------|:----------------------------------------------------------|
| 与         | a & b | 对于每一个比特位，对应的比特位都为1时，结果才为1，否则位0 |
| 或         | a \| b |  对于每一个比特位，对应的比特位至少一个为1时，结果才为1，否则位0  |
| 异或       | a ^ b  | 对于每一个比特位，对应的比特位有且只有一个为1，结果才为1，否则位0  |
| 非         |  ~ a  |   反转操作位的比特位，1 变 0，0 变 1  |
| 左移       | a << b |    a 的二进制向左移 b 位，左边用0填充  |
| 有符号右移 |   a >> b   | a 的二进制向右移 b 位，丢弃被移出的位 |
| 无符号右移 |   a >>> b    | 将二进制的表示向右移 b 位，丢弃被移出的位，并使用 0 填充左侧 |
### 2.1 与（&）
#### 运算步骤
1. 数字转换成二进制补码
2. 相同位进行比较（1&1 = 1, 否则为 0）
3. 如果计算结果为负，还要再做补码处理
#### 示例
``` js
10 & 3      // 2
14 & -13    // 2  
-12 & -5    // -16
```
### 2.2 或（|）
#### 运算步骤
1. 数字转换成二进制补码
2. 相同位进行比较（0 | 0 = 0, 否则为 1）
3. 如果计算结果为负，还要再做补码处理
#### 示例
``` js
10 | 3      // 11
10 | -3     // -1
-15 | -21   // -5
```
### 2.3 非（～）
#### 运算步骤
非 `~` 运算又称取反运算，十进制可视为
1. 取数字的负数
2. 减1
#### 示例
``` js
~ 1         // -2
~ -1        // 0
```
### 2.4 异或（^）
#### 运算步骤
1. 把两个数转换成2进制补码
2. 相同位置进行比较（必须是0和1或者1和0，结果才为1）
3. 如果结果为负，再取补码
#### 示例
``` js
10 ^ 3      // 9
10 ^ -3     // -9
-10 ^ -3    // 11
```
### 2.5 带符号左移（<<）
#### 运算步骤
1. 把数字转换成2进制补码
2. 左移指定位数，右边补0
3. 如果结果未负数，再取补码
#### 示例
``` js
1 << 2      // 00000001 左移2位, 即 00000100，结果为：4
-6 << 3     // 1010 << 3 等于 1010000，取补码，1110000 即：-48
```
### 2.6 带符号右移（>>）
#### 运算步骤
1. 取数字二进制补码
2. 右移指定位数，左边补位与符号位一致
3. 多余位被舍弃
4. 如果计算结果为负，再取补码
#### 示例
``` js
5 >> 1      // 0101 右移1位 0010，即 2
-3 >> 2     // 1101 右移两位 1001，即-1
```
### 2.7 无符号右移（>>>）
#### 运算步骤
1. 把数字转换成32位2进制补码
2. 连同符号位，右移动指定的位数
3. 向右被移出的位被丢弃，左侧用0填充
#### 示例
``` js
5 >>> 2      // 101 右移2位 001 即：1
-5 >>> 2     // 补码：11111111 11111111 11111111 11111011，右移：00111111 11111111 11111111 11111110
```
