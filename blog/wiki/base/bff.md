# BFF 与 Serverless
## 1. BFF
### 1.1 什么是BFF
BFF层是在后台服务与前端之间添加一层

主要作用是：**用户体验适配层和API聚合层**，主要负责快速跟进UI迭代，对后端接口服务进行组合、处理，对数据进行裁剪、格式化、聚合等等，主要技术栈是 Node + GraphQL
### 1.2 BFF的优势
- 降低沟通成本
- 多端应用适配
### 1.3 BFF的痛点
- 重复开发
- 维护问题：需要维护各种BFF应用，还需要考虑并发问题
- 链路复杂：流程变得繁琐
- 资源浪费：多个一层，资源占用就成了问题

## 2. Serverless
### 2.1 什么是 Serverless
Serverless = Faas(Function as a service) + Baas(Backend as a service)
### 2.2 云函数（Faas）
是服务商提供的一个平台，提供给用户开发、运行、管理这些函数功能，而无需搭建和维护基础框架，是一种事件驱动，由消息触发的函数服务
### 2.3 后端即服务（Bass）
后端即服务：包含了后端服务组件，是基于 API 的第三方服务，用于实现应用程序中的核心功能，包含常用的数据库、对象存储、消息队列、日志服务等

### 2.5 优势
- 环境统一
- 按需计费
- 丰富的SDK：有完善的配套服务
- 弹性伸缩：不需要预估流量、关心资源利用率、备份容灾、机器扩容，可根据流量动态改变
### 2.6 缺点
- 云厂商绑定：通常会和厂商的其他云产品绑定
- 不适合长时间任务
- 冷启动时间：函数运行时，执行容器和环境需要一定的时间，会增加 HTTP 请求时间
- 调试与测试：开发测试时会产生费用
### 2.7 应用场景
- 负载有波峰波谷：serverless 不用为了波峰去准备，支持弹性扩容
- 定时任务：服务空闲时间用来处理批量数据，生成数据报表
- 小程序开发（云开发）：微信小程序世纪开发过程获取 openid 等操作比较繁琐，这些都可交给云函数去处理
