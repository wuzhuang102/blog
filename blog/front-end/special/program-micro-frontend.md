# 技术解决方案 --- 微前端

## 微前端概念
### 1. 定义
微前端（Micro-Frontend），是将微服务（Micro-Services）理念用于前端技术后的相关实践，使得一个前端项目能够经由多个团队独立开发以及独立部署。
### 2. 特点
1. 各开发团队可自行选择技术栈，不受同一项目中其他团队影响
2. 各个交付产物都可以独立使用，避免和其它交付产物耦合
3. 各个交付产物中的样式不会污染其它组件
4. 各个交付产物可以自由使用浏览器原生API，而非要求使用封装后的API
### 3. 微前端要解决的问题
1. 微应用的注册、异步加载和声明周期管理
2. 微应用之间、主从之间的消息机制
3. 微应用之间的安全隔离措施
4. 微应用的框架无关、版本无关
5. 微应用之间、主从之间的公共依赖的库、业务逻辑(utils)以及版本怎么管理
6. 微应用独立调试、和主应用联调的方式。快速定位问题（发射问题）
7. 微应用的发布流程
8. 微应用的打包优化问题
9. 微应用专有打包云场景出包方案
10. 渐进式升级：用微应用方案平滑重构老项目



## 微前端方案
1. iframe 嵌入。
    - iframe缺点太多，可以舍弃
2. 创建 npm 私有仓库，动态的去维护。
    - 缺点：更新的业务代码，需要持续的去 update
3. system.js 动态引入。webpack 打出支持 system.js 导入的代码
4. 社区成熟解决方案： qiankun、single-spa
