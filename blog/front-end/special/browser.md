# 前端原理 --- 浏览器专题

## 1. 浏览器中的进程与线程

-   浏览器中的进程
    1. **浏览器主进程**：负责浏览器地址栏、书签栏、前进后退等
    2. **渲染进程**：一个 tab 内关于网页呈现的所有事情
    3. **GPU 进程**：GPU 相关任务
    4. **网络进程**：负责浏览器底层的一些不可见操作，网络请求、文件获取等
    5. **其它插件进程**：flash 等插件
-   浏览器的渲染进程中拥有的线程

    1. **GUI 渲染线程**
        - 负责页面渲染，解析 HTML、CSS、DOM、Render Tree、Layout、Painting
        - Repaint、Reflow 时也触发
        - GUI 线程与 JS 线程是互斥的，JS 引擎线程优先级高于 GUI 线程
    2. **JS 引擎线程**
        - 负责处理和解析 JS 脚本
        - JS 引擎一直等待任务队列中的任务，render 进程中永远只会有一个 JS 线程在运行
        - 注意：GUI 线程与 JS 引擎线程是互斥的，JS 执行时间长，页面就会渲染不连贯、卡顿
    3. **事件触发线程**
        - 归属于浏览器而不是 JS 引擎
    4. **定时器线程**
        - `settInterval` `setTimeout`所在的线程
        - 浏览器计数不是由 JS 引擎完成的（如果是单线程的 JS 来完成，那么会卡死的）
    5. **异步 HTTP 线程**
        - XMLHttpRequest 在连接后通过浏览器开启一个线程请求
        - 状态变更时，如果有回调函数，一步线程就会产生状态变更事件，将这个回调放入事件队列中，由 JS 引擎执行

## 2. 浏览器解析 HTML 的过程

1. 构建 DOM 树
2. 样式计算，构建 CSSOM
3. 布局
    - 可见元素创建布局树 LayoutTree
4. 分层
    - 拥有层叠上下文、需要裁减的地方都会创建一个单独的图层
5. 图层绘制
6. 合成线程分块
    - 合成线程将图层分为图块
7. 栅格化
    - 将合成线程中形成的图块转换为位图
    - 栅格化过程会使用 GPU 加速生成，生成的位图也保存在GPU中
8. 合成显示
    - 所有图块栅格化完成，合成线程就会把内容提交给浏览器中的一个 viz 组件，它会将这些内容绘制到内存中，最后将内存显示在屏幕上
