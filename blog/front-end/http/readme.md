# HTTP 简介
![http protocol](/http/http-protocol.jpg)

## 什么是HTTP协议
- HTTP是超文本传输协议(Hypertext Transfer Protocol)，它是为web浏览器与web服务器之间通信设计的，但也可用于其他，位于 TCP/IP 协议 应用层
- HTTP协议是由客户机到服务器的请求 (Request) 和 从服务器到客户机的响应 (Response) 进行约束和规范

### TCP/IP 协议
1. 应用层
    - 为用户提供服务，如HTTP协议、FTP协议、SSH协议、SSH协议、DNS协议、SMTP协议等
2. 传输层
    - 给应用层实体提供端到端的通信，保证数据的传输及完整
    - 该层定义了两个主要协议：传输控制协议（TCP）和 用户数据报协议（UDP）
3. 网络层
    - 主要解决主机到主机的通信问题
4. 网络接口层
    - 负责监视数据在主机和网络之间测交换
### ISO 协议
1. 应用层
    - 为用户提供服务，如HTTP协议、FTP协议、SSH协议、SSH协议、DNS协议、SMTP协议等
2. 表示层
    - 用于处理在两个通信系统中交换信息的表示方法，主要包括数据格式变换、数据加密解密、数据压缩与恢复等
3. 会话层
    - 负责两个节点传输连接，以便确保点到点传输不中断。管理数据交换
    - 封装HTTPS密钥交换
4. 传输层
    - 该层定义了两个主要协议：传输控制协议（TCP）和 用户数据报协议（UDP）
5. 网络层
    - 也叫IP层，IP在网络层分配。
6. 数据链路层
    - 涉及网卡（MAC地址）
7. 物理层
    - 实际上指的是硬件、网卡等

## TCP的三次握手与四次挥手
为保证传输的可靠性，三次握手是理论的最小值
| 字段 | 含义                                                                      |
|:-----|:--------------------------------------------------------------------------|
| SYN  | 请求建立连接，并在其序列号的字段进行序列号的初始值设置。建立连接，设置为1 |
| ACK  | 确认是否有效，一般设置为1｜                                               |
| RST  | 对方要求重新连接，复位                                                    |
| FIN  | 希望断开连接                                                              |
### 三次握手
1. 客户端尝试连接服务器，向服务器发送 syn(syn = x) 包，进入 SYN_SEND 状态等待服务器确认
2. 服务器接收到客户端 syn 包并确认(ack = x + 1)，同时向客户端发送一个 syn (syn = y) 包，即： syn + ack，此时服务器进入 SYN_RECV状态，告知客户端：收到请求、随时接收。
3. 客户端收到服务器的 syn 包，向服务器发送确认包 ack (ack = y + 1)，发送完毕，客户端进入 ESTBLISHED 状态，服务器接收到亦如此，三次握手完成
### 四次挥手
1. **第一次**：客户端发出连接释放报文，并停止发送数据。FIN = 1,序列号为 seq = u, 此时客户端进入 FIN-WAIT-1 状态
2. **第二次**：服务器收到释放报文，发出确认报文，ACK = 1, ack = u + 1 ,并带上自己的序列号 seq = v，服务端进入 CLOSE-WAIT 状态
3. 客户端收到服务器确认请求后，此时客户端就进入FIN—WAIT-2 状态，等待服务器发送连接释放报文
4. **第三次**：服务器发送一个 FIN = 1，ack = u + 1 ，此时服务器进入 LAST-ACK状态，等待服务器确认.假定此时序列号为 seq = w
5. **第四次**：客户端收到释放报文，必须发出确认，ACK=1，ack = w +1，客户端进入 TIME-WAIT 状态此时还没有释放TCP连接，必须经过 2**MSL(最长报文寿命时间),客户端撤销TCP才进入CLOSED
6. 服务端接收到客户端发送的确认，直接进入CLOSEd状态

## HTTP的工作过程
1. 客户机与服务器建立连接
2. 建立连接后，客户机发送一个请求给服务器，请求格式为：统一资源标识符(URL)、协议版本号，后面是MIME信息包括请求修饰符、客户机信息和可能的内容。
3. 服务器接收到请求后，给予相应的响应信息，其格式为状态行，包括协议的版本号、一个成功或错误的代码，后面是MIME信息包括服务器信息、实体信息和可能的内容。
4. 客户机接收服务器所返回的信息通过浏览器显示在用户的显示屏上，然后与服务器断开连接。


以上任何一步出错，产生的错误信息将返回到客户端。
