# 移动端专题
## 1. RN、Weex、Flutter 的异同与优缺点
- **RN**
    - 优点
        - 使用 JS 编写原生应用，上手快
        - 基础 UI 组件 与 原生应用完全一致
        - Hot Reloading
        - 兼容原生
    - 缺点
        - 未完全跨品台，有时候需要针对特定平台写兼容代码
        - 与原生消息同步时有性能消耗
        - 动画效率低下
        - 平台的兼容代码要考虑
- **Weex**
    - 优点
        - 使用 JS 编写原生应用，上手快
        - **轻量**：默认打包只包含业务代码，基础库在 Weex SDK 中
        - 支持 web
    - 缺点
        - 文档不友好，坑多
        - 社区不活跃
        - 其它两者的缺点它都有
- **Flutter**
    - 优点
        - **Dart语言优势**：开发阶段采用 JIT 模式，避免重复编译，节省开发时间；发布时通过 AOT 生成高效的 ARM 代码
        - **高性能**：使用自己的渲染引擎绘制 UI（没有混合APP的信息同步开销），UI性能可与原生比肩
        - **稳定性**：UI 不依赖 CSS 等外部解释器，保证了跨平台的一致性
        - **优秀的路由**：反向传值设计，返回当前页面可以更方便的获取传入的参数
        - **优秀的动画**：自绘引擎性能比混合的好
    - 缺点
        - 本质没有脱离原生，开发者需要具备基础的原生开发能力
        - 组合而非继承
        - UI 控件 API 不友好
        - 资源管理困难
        - 第三方 SDK 兼容匹配困难，需要原生集成
